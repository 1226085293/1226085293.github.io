你有遇到过需要提交 0 修改的 meta 文件的情况吗？在不同系统下，有可能会造成项目的 meta 文件换行符不一致，例如*windows 的\r\n 和 linux 的\n*

### 在工作区使用 CRLF，使用 git commit 提交的时候 git 帮你把所有的 CRLF 转换为 LF。

git config --global core.autocrlf true

### 在工作区使用 LF

git config --global core.autocrlf input

### 避免文件中有混用换行符

git config --global core.safecrlf true

---

这里我使用的是*git config --global core.autocrlf input*

也可以不需要上面的命令使用 [git 配置文件](https://forum.cocos.org/uploads/short-url/z4laNgrsNh8pScMLAZ6veYI6GDd.zip)，放在仓库根目录
